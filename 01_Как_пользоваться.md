# discord-session-transcriber — Как пользоваться

## 0) Установка (один раз)

Нужен **Python 3.11+ x64** в PATH и **ffmpeg в `PATH`** (или в `tools/ffmpeg/bin`).

```bash
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate      # Linux/macOS
pip install -e .
```

ASR-бэкенд (faster-whisper / GigaAM и т.п.) приложение скачивает само в изолированный
каталог при первом запуске — отдельно ставить PyTorch/WhisperX не нужно.

## 1) Подготовка сессии

В папке сессии должны быть раздельные дорожки Craig (обычно `1-*.flac`, `2-*.flac` …).
Файлы `craig.flac`/`craig.json` — общий микс, они автоматически исключаются из обработки.

Опционально: `speaker_map.json` в папке сессии для подстановки имён:

```json
{
  "1-vivienen": {"player": "Катя", "character": "Одетт", "role": "PC"},
  "2-v_vladimir": {"player": "Вова", "character": "ГМ", "role": "GM"}
}
```

Ключ — имя файла без расширения (например, `1-vivienen` для `1-vivienen.flac`).

## 2) Запуск

```bash
python -m ui
```

Откроется QML-GUI. В нём:
- **Папка сессии** — путь к папке с .flac файлами;
- **model**: `large-v3` (по умолчанию);
- **compute**: `float16` (по умолчанию, для GPU);
- **beam**: `10` (по умолчанию);
- **Нарезать на чанки**: включено по умолчанию;
- Нажать **«Запустить»**.

Если JSON-файлы от WhisperX уже есть — поставь галочку «Только склейка» и WhisperX запускаться не будет, только merge + chunking.

## 3) Что получаем

В папке сессии:
- **`merged.txt`** — единый текст по спикерам, отсортированный по времени.
- **`chunks/0001.txt …`** — чанки с overlap (с маркерами `=== CHUNK START/END ===`).
- **`chunks/000_context.txt`** — контекст (список спикеров, заметки).
- **`chunks/manifest.json`** — метаданные о чанках.

## 4) Работа с LLM

1. Прогнать chunks по порядку через промпт **`prompts/01_raw_to_transcript.md`** → получить литературную стенограмму.
2. Из стенограммы сделать художественный журнал через промпт **`prompts/02_transcript_to_journal.md`**.

## 5) CLI-режим (продвинутый)

С аргументами `python -m ui` уходит в headless-обработку (без GUI):

```cmd
venv\Scripts\python.exe -m ui "D:\games\session_2025-07-11" --merge --chunk
```

Все флаги: `--model`, `--language`, `--device`, `--compute_type`, `--beam_size`, `--merge`, `--chunk`, `--merge_only`, `--dry_run` и т.д. Подробнее: `python -m ui --help`.
