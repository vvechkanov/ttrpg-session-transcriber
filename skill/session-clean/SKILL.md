# Session Clean — Очистка транскрипта

## Обзор

Скилл работает в двух режимах:
1. **Очистка чанка** — один сырой чанк → один clean файл
2. **Склейка** — несколько clean файлов → один session_clean.txt

### Workflow (всего N+1 чатов, где N = количество чанков)

```
Чат 1: 0001.txt + manifest.json + 000_context.txt → clean_0001.txt
Чат 2: 0002.txt + manifest.json + 000_context.txt → clean_0002.txt
...
Чат N: 000N.txt + manifest.json + 000_context.txt → clean_000N.txt
Чат N+1: все clean файлы + manifest.json          → session_clean.txt
```

Каждый чат — отдельный, чтобы не забивать контекст.

---

## Режим 1: Очистка чанка

### Входные данные
- Файл чанка (0001.txt, 0002.txt и т.д.)
- manifest.json (метаданные)
- 000_context.txt (спикеры)

### Уровень чистки: МИНИМАЛЬНЫЙ
Это чистовая стенограмма, НЕ литературная обработка.

### Что УБИРАТЬ
1. **Метаданные чанков** — `=== CHUNK START/END ===`, строки `chunk:`, `source:`, `paragraphs:`
2. **Мета-разговоры вне игры — УБИРАТЬ ПОЛНОСТЬЮ:**
   - Поиск заметок ГМ-ом: "Сейчас, секунду, я найду...", "Дайте мне минутку", "Я потерял эти данные"
   - Технические обсуждения: "Крейг", "дискорд", "микрофон", "запись", "бот"
   - Приветствия бота: "Здравствуй, бот"
   - Целые реплики-паузы ГМ: если реплика ЦЕЛИКОМ состоит из мета-текста — убрать всю реплику
   - Если мета-текст ВНУТРИ содержательной реплики — убрать мета-часть, оставить содержательную
3. **Повторы-артефакты** — "во, во, во, во", двойные слова подряд от Whisper
4. **Пустые реплики** — `Спикер: Угу.` или `Спикер: Ага.` если не несут смысла
5. **Артефакты Whisper** — "Субтитры сделал...", "Продолжение следует..." (вставки Whisper)
6. **Послесессионные разговоры** — дебриф ГМ о механиках, прощания, планы на следующую сессию — УБИРАТЬ ПОЛНОСТЬЮ

### Что СОХРАНЯТЬ
1. **Все метки спикеров** — формат `Спикер:` или `Спикер (Персонаж):`
2. **Всю игровую информацию** — описания, диалоги, NPC, локации, предметы
3. **Механики** — числа, уровни, DC, HP, проверки, названия заклинаний и способностей
4. **Разговорный тон** — речь остаётся живой, не литературной
5. **Короткие реакции** если несут контекст (согласие по важным решениям, смех в RP)
6. **OOC-обсуждения тактики и правил** если касаются игровых решений

### Процесс обработки

1. Прочитай `000_context.txt` — узнай спикеров и маппинг
2. Прочитай чанк целиком
3. Примени правила чистки
4. Объедини разорванные Whisper-ом реплики одного спикера (когда подряд идут 2+ реплики одного человека — склеить в одну, если это одна мысль)
5. Исправь очевидные ошибки Whisper в именах PF2e (Чикек→Ачакек, Мэри→Маэри и т.п.)
6. Выдай результат как файл `clean_NNNN.txt`

### Выход
Файл `clean_NNNN.txt` — чистый текст чанка с метками спикеров.

## Режим 2: Склейка

### Входные данные
- Все clean файлы (clean_0001.txt, clean_0002.txt, ...)
- manifest.json (нужен для данных об оверлапах)

### Процесс склейки

Manifest содержит `overlap_paragraphs` для каждого чанка — количество параграфов
в начале чанка, которые дублируют конец предыдущего.

Но после чистки точные границы сместились, поэтому алгоритм:
1. Взять clean_0001.txt полностью
2. Для каждого следующего clean файла:
   - Найти последние 3-5 уникальных реплик из конца предыдущего файла
   - Найти эти же реплики в начале текущего файла (fuzzy match по содержанию)
   - Отрезать всё до точки совпадения
   - Приклеить оставшееся
3. Записать результат в `session_clean.txt`

### Выход
Файл `session_clean.txt` — полный чистый текст сессии без дубликатов.

## Важные заметки

### Имена и маппинг спикеров
Берётся из 000_context.txt. В чистом тексте сохраняем оригинальные метки.
Пример:
```
- Вова              → ГМ
- Катя (Агата)      → Агата
- Тоша (Лисса)      → Лисса
- Настя (Анжелика)  → Анжелика
```

### Частые ошибки Whisper в PF2e
- Чикек, Чекек, Очекек → Ачакек
- Мэри, Мария → Маэри (сестра Маэри)
- Лёд, Лёте, лед → Летте (или Лётте, если контекст подсказывает)
- Ирваэль — корректно
- Лизмагорт — корректно
- Богомолы, красные богомолы — корректно (Красные Мантисы)

### Кампания
Pathfinder 2e, мир Голарион. Если Whisper исковеркал название — исправлять по контексту.
