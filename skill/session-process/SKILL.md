# Session Process — История, саммари, Obsidian

## Обзор

Этот скилл берёт готовый очищенный транскрипт (session_clean.txt) и делает:
- **Шаг 2:** Подробная история сессии (нарратив)
- **Шаг 3:** Краткий пересказ (саммари)
- **Шаг 4:** Обновление Obsidian

Каждый шаг запускается отдельно по команде пользователя.

### Входные данные
- `session_clean.txt` — полный очищенный текст сессии (выход скилла session-clean)
- Опционально: `000_context.txt` — маппинг спикеров

---

## Шаг 2: История

### Цель
Нарративный пересказ сессии от третьего лица. Что произошло, в хронологическом порядке.

### Правила
- Markdown, заголовки по сценам/локациям
- Имена ПЕРСОНАЖЕЙ (не игроков): Агата, Лисса, Анжелика, Ирваэль (не Катя, Тоша, Настя)
- Вова → ГМ, его речь — описания мира, NPC, механики
- Описания окружения из речи ГМ — сохранять и развивать
- Диалоги NPC — можно пересказать или дать ключевые фрагменты
- Механики: упоминать только если повлияли на сюжет ("Агата провалила проверку Дипломатии")
- Боёвки: описать ход боя, ключевые моменты, итог. Не перечислять каждый бросок.
- Длина: подробная, но без дословного цитирования стенограммы

### Маппинг спикеров (из 000_context.txt)
```
Вова              → ГМ (описания, NPC, механики)
Катя (Агата)      → Агата / Адет (персонаж)
Тоша (Лисса)      → Лисса / Лётте / Ирваэль (персонаж)
Настя (Анжелика)  → Анжелика (персонаж)
```

### Выход
Файл `session_story.md`

---

## Шаг 3: Краткий пересказ

### Цель
Саммари сессии на 1-2 страницы для быстрого референса.

### Формат
- Markdown
- Структура:
  - Дата сессии
  - Участники (персонажи)
  - Ключевые события (буллеты)
  - Итоги
  - Открытые вопросы / зацепки
- Имена персонажей (не игроков)
- Без детальных механик и бросков

### Выход
Файл `session_summary.md`

---

## Шаг 4: Обновление Obsidian

### Цель
Обновить файлы в Obsidian-волте на основе сессии.

### Что обновляется
1. **Файл сессии** — создать/обновить в `Кампейны\Азланти\Сессии\`
2. **NPC-файлы** — если появились новые факты о NPC, обновить их файлы в `Кампейны\Азланти\NPC\`
3. **Квестовые файлы** — если квест продвинулся, обновить статус

### Правила работы с Obsidian
- **НИКОГДА не создавать новые файлы если существует старый** — редактировать существующие
- Использовать `Desktop Commander:edit_block` для правок
- Путь к волту: `C:\DND\sync\Sync\Obsidian\PF2e\`
- Путь к кампании: `Кампейны\Азланти\`

### Процесс
1. Спросить пользователя, что именно обновить (или он сам скажет)
2. Найти существующие файлы через Obsidian MCP поиск
3. Прочитать текущее содержимое
4. Внести правки через edit_block
5. Отчитаться о сделанных изменениях

### Выход
Обновлённые файлы в Obsidian.
